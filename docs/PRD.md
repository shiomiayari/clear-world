# PRD（製品要件定義書） — Clear World

**バージョン:** v0.1
**作成日:** 2026-03-22
**ステータス:** Approved

---

## 1. 背景・課題

YouTube Shortsの無限スクロール設計はユーザーの意図しない長時間視聴を引き起こす。
既存の「スクリーンタイム」機能は数字を見せるだけで、行動変容につながらない。
罪悪感ではなく「美しい世界を守りたい」という内発的動機（損失回避）に訴える行動変容ツールが必要。

---

## 2. ターゲットユーザー

- YouTube Shortsを日常的に使い、「見すぎた」と感じた経験がある人
- 自己嫌悪に陥りやすく、強制的な制限より「自分で気づいて止める」を好む人
- 「昨日の自分より少し良くなった」という小さな成功体験を求めている人
- スマートフォンのホーム画面ウィジェットをよく使う人

---

## 3. ソリューション概要

| 要素 | 内容 |
|---|---|
| **水槽ウィジェット** | 透明度が「今日の自分」を映す鏡。昨日より視聴が少なければ澄み、多ければ濁る |
| **魚の生成** | 節約できた時間が「色鮮やかな魚」として水槽に加わる |
| **ポップアップ警告** | 連続20分・前日比悪化を検知して「水槽が濁り始めています」と表示 |

---

## 4. 成功指標（KPI）

| 指標 | 目標値 |
|---|---|
| 7日継続率 | 40%以上 |
| 平均日次視聴時間の削減率 | 使用開始から2週間で20%削減 |
| ウィジェット設置率 | インストールユーザーの60%以上 |

---

## 5. 計測ロジック仕様

### 時間帯ブロック定義

| ブロック名 | 時間帯 |
|---|---|
| 深夜（midnight） | 24:00〜翌6:00 |
| 朝（morning） | 6:00〜12:00 |
| 昼（afternoon） | 12:00〜18:00 |
| 夕方（evening） | 18:00〜24:00 |

### 基本ルール
- 視聴時間のリセット：毎日**朝6:00**
- 計測対象：YouTubeアプリ内のShortsコンポーネントが**画面に表示されている時間**
- ブラウザ版YouTube Shortsは対象外

---

## 6. 透明度ロジック

```
Δ = -(diff_seconds / 1800) × 10    // 30分差で±10%
新透明度 = 現在の透明度 + Δ
新透明度 = max(0, min(100, 新透明度))
```

| 状態 | 条件 |
|---|---|
| 完全に濁る（0%） | 1ブロックで3時間以上の視聴 |
| 完全に透明（100%） | 昨日比-80%を3ブロック連続で達成 |
| 初日スタート | 60% |
| 日次リセット後スタート | 50% |

---

## 7. 魚の仕様

| パラメータ | 決定要因 |
|---|---|
| 色の鮮やかさ | 節約時間が長いほど鮮やか |
| サイズ | 深夜=大、夕方=中、朝・昼=小 |
| 消滅条件 | 視聴時間が昨日より増えたとき |
| 消え方 | 泡になって消える |
| 誕生の通知 | なし（ウィジェットの変化で気づく） |

---

## 8. 通知・ポップアップ仕様

### 濁りの警告ポップアップ
- **トリガー①:** 連続視聴が20分を超えた瞬間（3分の中断でリセット）
- **トリガー②:** 現ブロックの視聴ペースが前日を上回った瞬間
- **表示内容:** 「水槽が濁り始めています。一度スマホを置いて、世界を澄ませませんか？」
- **表示形式:** アプリ内ポップアップ（OS通知ではない）

---

## 9. ウィジェット仕様

| サイズ | 表示内容 |
|---|---|
| 小（2×2） | 透明度の数値 + 水の色のみ |
| 中（4×2） | 水槽（透明度） + 魚の数 |
| 大（4×4） | 水槽全体 + 魚が泳いでいる様子 + 現ブロック名 |

- 透明度更新：**15分ごと**（バッテリーセーバー時は30分）
- 魚の更新：**ブロック終了時**

| 項目 | 仕様 |
|---|---|
| 保存場所 | 端末ローカルのみ（Room DB） |
