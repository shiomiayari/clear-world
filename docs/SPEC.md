**作成日:** 2026-03-22

## 機能① — 計測（Accessibility Service）

### 概要
AndroidのAccessibility Serviceを使い、YouTubeアプリ内でShortsコンポーネントが画面に表示されている時間のみを計測する。

### 入力・処理・出力

| 項目 | 内容 |
|---|---|
| **入力** | AndroidのAccessibility Event（画面変化イベント） |
| **処理** | Shortsコンポーネントの表示/非表示を検知し、時間を積算 |
| **出力** | 現在のブロックの累計視聴秒数（ローカルDB保存） |

### 検知ロジック

```
AccessibilityEvent を受信
  ↓
パッケージ名が "com.google.android.youtube" か？
  No → 無視
  Yes ↓
画面内に Shorts コンポーネントが存在するか？
  No → 計測停止・タイマーを止める
  Yes → 計測中でなければタイマー開始
```

### 連続視聴の判定

```
Shortsから離脱を検知
  ↓
3分間のバッファタイマーを開始
  ↓
3分以内に戻った → 連続とみなす（タイマー継続）
3分経過しても戻らない → 連続リセット（20分カウンターもリセット）
```

### 保存データ構造

```json
{
  "date": "2026-03-22",
  "block": "evening",
  "total_seconds": 1240,
  "sessions": [
    { "start": "18:05:10", "end": "18:22:30" },
    { "start": "18:30:00", "end": "18:45:00" }
  ]
}
```

### 濁り警告のトリガー判定

| トリガー | 条件 |
|---|---|
| 連続視聴超過 | 連続視聴が1200秒（20分）を超えた瞬間 |
| 前日比悪化 | 現ブロックの累計が前日同ブロックの累計を超えた瞬間 |

### エッジケース

| ケース | 対応 |
|---|---|
| 日付をまたいで視聴（例：23:50〜0:10） | 深夜ブロックと翌朝ブロックに分割して記録 |
| 機内モード・オフライン | 影響なし（ローカル計測のため） |
| YouTubeアプリのアップデートでコンポーネント名が変わる | 検知対象コンポーネント名を設定ファイルで管理し、アップデートで対応 |
| Accessibility Serviceが途中でOFFにされた | 計測停止・ホーム画面に「計測が止まっています」バナーを表示 |

## 機能② — 透明度計算エンジン

### 概要
機能①が記録した視聴時間をもとに、水槽の透明度（0〜100%）をリアルタイムに計算する。

### 入力・処理・出力

| 項目 | 内容 |
|---|---|
| **入力** | 今日の各ブロック累計秒数・前日の各ブロック累計秒数 |
| **処理** | ブロックごとの昨日比を計算し、透明度に加減算 |
| **出力** | 現在の透明度（0〜100の整数値） |

### 計算ロジック

```
① 現ブロックの差分を計算
   diff_seconds = 今日の累計 - 昨日の累計

② 差分を30分（1800秒）単位で10%に換算
   Δ = -(diff_seconds / 1800) × 10
   ※ 昨日より多い → マイナス（濁る）
   ※ 昨日より少ない → プラス（澄む）

③ 透明度を更新
   新透明度 = 現在の透明度 + Δ

④ 上限・下限でクランプ
   新透明度 = max(0, min(100, 新透明度))
```

### 計算例

| 昨日 | 今日 | Δ | 結果（現在70%の場合） |
|---|---|---|---|
| 30分 | 60分 | -10% | 60% |
| 60分 | 30分 | +10% | 80% |
| 0分 | 180分（3時間） | -100% → 上限で0% | 0%（完全に濁る） |

### 特殊ケース

| ケース | 処理 |
|---|---|
| **初日ユーザー** | 透明度60%からスタート。前日データは全ブロック0秒とみなす |
| **完全透明の条件** | 昨日比-80%を3ブロック連続で達成したとき100%に強制セット |
| **6:00リセット時** | 透明度は引き継がず50%からスタート |

### 更新タイミング

| タイミング | 処理 |
|---|---|
| 15分ごと（バックグラウンド） | 透明度を再計算してウィジェットに反映 |
| Shortsを閉じた直後 | 即時再計算（ポップアップ判定のため） |
| アプリを開いたとき | 即時再計算 |

### 透明度と水の色の対応

| 透明度 | 水の色 | 状態の名前 |
|---|---|---|
| 100% | 透き通った青 | 澄み渡っている |
| 75〜99% | 薄い青緑 | きれい |
| 50〜74% | くすんだ緑 | 少し濁っている |
| 25〜49% | 茶色がかった緑 | 濁っている |
| 0〜24% | 暗い茶色 | ひどく濁っている |

## 機能③ — 魚の生成・消滅（Magenta.js）

### 概要
ブロック終了時に昨日比を判定し、節約できていれば魚を生成。Magenta.jsを使って「世界に1匹だけの魚」を生成する。

### 生成トリガー

```
ブロック終了（12:00 / 18:00 / 24:00 / 6:00）
  ↓
今日の累計 < 昨日の累計 か？
  No  → 魚は生まれない
  Yes ↓
節約秒数・ブロック種別を Magenta.js に渡す
  ↓
魚データを生成してローカルDBに保存
  ↓
ウィジェットを更新
```

### Magenta.jsへの入力パラメータ

| パラメータ | 値の範囲 | 魚への影響 |
|---|---|---|
| `saved_seconds` | 0〜10800（3時間） | 色の鮮やかさ（長いほど鮮やか） |
| `block_type` | morning / afternoon / evening / midnight | 魚のサイズ |

### ブロック別サイズ

| ブロック | サイズ | 理由 |
|---|---|---|
| 深夜（midnight） | 大 | 深夜の誘惑に勝つのが最も難しい |
| 夕方（evening） | 中 | ゴールデンタイムで意志力が必要 |
| 朝（morning） | 小 | 比較的コントロールしやすい時間帯 |
| 昼（afternoon） | 小 | 比較的コントロールしやすい時間帯 |

### 魚データ構造

```json
{
  "id": "fish_20260322_evening",
  "born_at": "2026-03-22T18:00:00",
  "block": "evening",
  "saved_seconds": 1800,
  "size": "medium",
  "color_params": { "hue": 180, "saturation": 0.85 },
  "melody_seed": 42,
  "is_alive": true
}
```

### 消滅ロジック

```
ブロック終了時、今日の累計 > 昨日の累計
  ↓
該当ブロックに生きている魚が存在するか？
  No  → 何もしない
  Yes ↓
is_alive を false に更新
  ↓
ウィジェットで泡アニメーション再生
  ↓
魚はDBに残る（履歴として）が水槽には表示されない
```

### エッジケース

| ケース | 対応 |
|---|---|
| 同じブロックで複数回の生成・消滅が起きる | 1ブロックにつき魚は最大1匹 |
| Magenta.jsの生成に失敗した | デフォルトパラメータの魚をフォールバックとして使用 |
| 初日で昨日データが0秒 | 少しでも視聴したら前日比マイナスにならないため魚は生まれない |

## 機能④ — ウィジェット表示

### 概要
OSのウィジェット機能（Android AppWidget）を使い、アプリを開かなくても現在の透明度・魚をホーム画面に表示する。

### ウィジェットサイズ別の表示内容

| サイズ | 表示内容 |
|---|---|
| **小（2×2）** | 透明度の数値 + 水の色のみ |
| **中（4×2）** | 水槽（透明度） + 魚の数 |
| **大（4×4）** | 水槽全体 + 魚が泳いでいる様子 + 現ブロック名 |

### 更新ロジック

```
【透明度の更新】15分ごと
  AlarmManager が起動
    ↓
  透明度計算エンジンを呼び出し
    ↓
  ウィジェットのRemoteViewsを更新

【魚の更新】ブロック終了時のみ
  魚の生成・消滅処理が完了
    ↓
  ウィジェットの魚レイヤーを更新
    ↓
  泡アニメーション（消滅時）を再生
```

### タップ時の動作

すべてのサイズでタップするとアプリのホーム画面（水槽画面）を開く。

### エッジケース

| ケース | 対応 |
|---|---|
| Accessibility Serviceが停止している | ウィジェットに「計測が止まっています」バナーを表示 |
| ウィジェット追加直後（データなし） | 透明度60%・魚なしで初期表示 |
| バッテリーセーバーモード | 更新頻度を15分→30分に自動切替 |
| 複数のウィジェットを同時に配置 | 全サイズが同じデータを参照し、同時に更新される |

## ローカルDB テーブル定義（全機能共通）

```sql
-- 視聴記録
CREATE TABLE viewing_records (
    id          TEXT PRIMARY KEY,  -- "2026-03-22_evening"
    date        TEXT NOT NULL,
    block       TEXT NOT NULL,
    total_sec   INTEGER NOT NULL DEFAULT 0,
    sessions    TEXT NOT NULL DEFAULT '[]'  -- JSON配列
);

-- 魚
CREATE TABLE fish (
    id          TEXT PRIMARY KEY,  -- "fish_20260322_evening"
    born_at     TEXT NOT NULL,
    block       TEXT NOT NULL,
    saved_sec   INTEGER NOT NULL,
    size        TEXT NOT NULL,     -- "small" / "medium" / "large"
    color_hue   INTEGER NOT NULL,
    color_sat   REAL NOT NULL,
    melody_seed INTEGER NOT NULL,
    is_alive    INTEGER NOT NULL DEFAULT 1  -- 1=生存, 0=消滅
);

-- 透明度
CREATE TABLE transparency (
    date        TEXT NOT NULL,
    block       TEXT NOT NULL,
    value       INTEGER NOT NULL,  -- 0〜100
    PRIMARY KEY (date, block)
);
```
